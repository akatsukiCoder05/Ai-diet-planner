# 🚀 Backend Server Setup Guide

## Overview
This backend server provides PDF generation and diet plan management features for the AI Diet Planner application.

## Features

### 1. **PDF Generation** 📄
- Converts diet plan text to beautifully formatted PDF
- Includes user information section
- Proper text wrapping and pagination
- Custom styling with colors matching the app theme
- Automatic download with timestamp

### 2. **Diet Plan Management** 💾
- Save diet plans with user info
- View history of saved plans
- Retrieve specific plan by ID
- Delete saved plans
- In-memory storage (can be upgraded to database)

### 3. **Health Check** ✅
- Server status endpoint for monitoring

## Installation

### Step 1: Install Dependencies
```bash
npm install
```

This will install:
- `express` - Web server framework
- `cors` - Cross-origin resource sharing
- `pdf-lib` - PDF generation library
- `concurrently` - Run multiple commands simultaneously

### Step 2: Verify Installation
Check that these packages are in your `package.json`:
```json
{
  "dependencies": {
    "express": "^4.18.2",
    "cors": "^2.8.5",
    "pdf-lib": "^1.17.1"
  },
  "devDependencies": {
    "concurrently": "^8.2.2"
  }
}
```

## Running the Server

### Option 1: Run Server Only
```bash
npm run server
```
Server will start on http://localhost:5000

### Option 2: Run Frontend + Backend Together
```bash
npm run dev:all
```
This runs both:
- Frontend on http://localhost:5173
- Backend on http://localhost:5000

### Option 3: Run Separately (Two Terminals)

**Terminal 1 (Frontend):**
```bash
npm run dev
```

**Terminal 2 (Backend):**
```bash
npm run server
```

## API Endpoints

### 1. Health Check
```http
GET /api/health
```

**Response:**
```json
{
  "status": "ok",
  "message": "Server is running"
}
```

### 2. Generate PDF
```http
POST /api/generate-pdf
Content-Type: application/json
```

**Request Body:**
```json
{
  "dietPlan": "TITLE: Weight Loss Plan\n\nBMI:\n- value: 22.8...",
  "userInfo": {
    "age": "28",
    "gender": "Male",
    "activityLevel": "Active (Moderate)",
    "weight": "70",
    "goal": "Lose Weight"
  }
}
```

**Response:**
- Content-Type: `application/pdf`
- Downloads file: `diet-plan.pdf`

### 3. Save Diet Plan
```http
POST /api/save-plan
Content-Type: application/json
```

**Request Body:**
```json
{
  "dietPlan": "...",
  "userInfo": {...},
  "parsedData": {...}
}
```

**Response:**
```json
{
  "success": true,
  "message": "Diet plan saved successfully",
  "id": 1703123456789
}
```

### 4. Get History
```http
GET /api/history
```

**Response:**
```json
{
  "success": true,
  "history": [
    {
      "id": 1703123456789,
      "timestamp": "2024-12-21T10:30:00.000Z",
      "userInfo": {...},
      "bmi": "22.8",
      "calories": "2450"
    }
  ]
}
```

### 5. Get Specific Plan
```http
GET /api/history/:id
```

**Response:**
```json
{
  "success": true,
  "plan": {
    "id": 1703123456789,
    "timestamp": "2024-12-21T10:30:00.000Z",
    "userInfo": {...},
    "dietPlan": "...",
    "parsedData": {...}
  }
}
```

### 6. Delete Plan
```http
DELETE /api/history/:id
```

**Response:**
```json
{
  "success": true,
  "message": "Diet plan deleted successfully"
}
```

## Troubleshooting

### Port Already in Use
```
Error: listen EADDRINUSE: address already in use :::5000
```

**Solution:** Change the port in `server.js`:
```javascript
const PORT = process.env.PORT || 5001; // Changed to 5001
```

### CORS Errors
```
Access to fetch at 'http://localhost:5000/api/...' from origin 'http://localhost:5173' has been blocked
```

**Solution:** CORS is already enabled in `server.js`. If issues persist, check:
```javascript
app.use(cors({
  origin: 'http://localhost:5173',
  methods: ['GET', 'POST', 'DELETE'],
  credentials: true
}));
```

### PDF Generation Fails
```
Error generating PDF
```

**Solution:** 
1. Check that `pdf-lib` is installed: `npm list pdf-lib`
2. Verify the request body contains `dietPlan` text
3. Check server console for detailed error messages

### Dependencies Not Found
```
Error: Cannot find module 'express'
```

**Solution:**
```bash
npm install express cors pdf-lib
```

## Upgrading to Database

The current implementation uses in-memory storage. To upgrade to a database:

### MongoDB Example
```javascript
import mongoose from 'mongoose';

const dietPlanSchema = new mongoose.Schema({
  timestamp: Date,
  userInfo: Object,
  dietPlan: String,
  parsedData: Object,
});

const DietPlan = mongoose.model('DietPlan', dietPlanSchema);

// Save
const newPlan = new DietPlan({
  timestamp: new Date(),
  userInfo: req.body.userInfo,
  dietPlan: req.body.dietPlan,
  parsedData: req.body.parsedData,
});
await newPlan.save();
```

## Production Deployment

### Environment Variables
Create a `.env` file:
```env
PORT=5000
NODE_ENV=production
ALLOWED_ORIGINS=https://your-frontend-domain.com
```

### Security Enhancements
```javascript
import helmet from 'helmet';
import rateLimit from 'express-rate-limit';

app.use(helmet());

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limit each IP to 100 requests per windowMs
});

app.use('/api/', limiter);
```
